## 📍 使用场景导航

### 场景1：我是新用户，想快速体验

1. 阅读 [README.md](README.md) 了解项目
2. 使用 [镜像快速参考.md](镜像快速参考.md) 快速启动
3. 或者运行自动化脚本：
   ```bash
   ./scripts/setup.sh
   ```

### 场景2：我想使用预构建的Docker镜像

1. 阅读 [DOCKER镜像使用指南.md](DOCKER镜像使用指南.md)
2. 下载 [docker-compose.ghcr.yml](docker-compose.ghcr.yml)
3. 修改配置中的用户名
4. 运行：`docker-compose -f docker-compose.ghcr.yml up -d`

### 场景3：我想本地开发和调试

1. 阅读 [README.md](README.md) 的快速开始部分
2. 使用 [docker-compose.dev.yml](docker-compose.dev.yml) 启动开发环境
3. 查看 [backend/README.md](backend/README.md) 了解后端开发
4. 修改代码，服务会自动重载

### 场景4：我想部署到生产环境

1. 详细阅读 [DEPLOYMENT.md](DEPLOYMENT.md)
2. 配置环境变量（参考 [backend/.env.example](backend/.env.example)）
3. 选择部署方式：
   - 使用预构建镜像：[docker-compose.ghcr.yml](docker-compose.ghcr.yml)
   - 本地构建：[docker-compose.yml](docker-compose.yml)
4. 配置SSL证书和域名
5. 设置定期备份

### 场景5：我想了解系统架构

1. 阅读 [ARCHITECTURE.md](ARCHITECTURE.md) 了解整体架构
2. 阅读 [PROJECT_SUMMARY.md](PROJECT_SUMMARY.md) 了解改造过程
3. 查看 [CHANGES.md](CHANGES.md) 了解具体变更

### 场景6：我遇到了问题

1. 查看 [QUICK_START.md](QUICK_START.md) 的故障排除章节
2. 查看 [DOCKER镜像使用指南.md](DOCKER镜像使用指南.md) 的故障排除
3. 查看 [使用说明.md](使用说明.md) 的常见问题
4. 查看容器日志：`docker-compose logs -f`

### 步骤2：查看日志

```bash
# 查看所有服务日志
docker-compose logs -f

# 查看特定服务
docker-compose logs -f backend
```

### 步骤3：检查健康状态

```bash
# 检查容器状态
docker-compose ps

# 检查API健康
curl http://localhost:8000/api/health
```
